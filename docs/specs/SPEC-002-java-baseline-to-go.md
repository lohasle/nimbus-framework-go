# SPEC-002 Java 底座迁移到 Go

## 状态

- 当前：进行中
- 源工程：`nimbus-framework/backend`
- 目标工程：`nimbus-framework-go/backend`
- 数据库基线：MySQL 8.4

## 目标

Go 版不是缩减功能的演示工程，而是 Java Nimbus 底座的 Go 实现。前端接口契约、数据库语义、权限与菜单行为应保持兼容，使同一套 Nimbus 运营后台能够正常登录并使用已迁移的通用中心。

## 工程结构

```text
nimbus-framework-go/
├── frontend/                 # Nimbus Vue 运营后台
└── backend/
    ├── cmd/server/           # 模块化单体启动入口
    ├── internal/platform/    # 配置、数据库、HTTP、中间件、路由等公共能力
    ├── internal/modules/
    │   ├── system/           # 账号、组织、角色、权限、租户、字典与审计
    │   ├── infra/            # 配置、文件、任务、日志等基础设施能力
    │   ├── member/           # 会员、等级、标签、积分等会员能力
    │   ├── pay/              # 支付应用、渠道、订单、退款与钱包能力
    │   ├── application/      # 自有预留模块，本阶段仅 Health
    │   ├── im/               # 自有预留模块，本阶段仅 Health
    │   └── app/              # 自有编排模块，本阶段仅 Health
    └── docs/                 # Swagger / OpenAPI 生成物
```

当前采用 GORM 显式模型迁移与幂等种子数据初始化，不额外维护一套重复的 SQL migration 文件。

## 迁移边界

### 必须迁移

1. `system`：Java 底座已有的后台用户、部门、岗位、角色、菜单、租户、字典、参数和日志等通用能力。
2. `infra`：Java 底座已有的配置、文件、任务、API 日志等通用能力；依赖外部中间件的能力允许通过配置启用。
3. `member`：Java 底座已有的会员、等级、标签、分组、积分、签到等能力。
4. `pay`：Java 底座已有的支付应用、渠道、订单、退款、转账和钱包等能力；真实第三方通道必须使用显式配置。
5. 保持前端使用的 `/admin-api` 响应结构、分页结构、认证头、租户头和菜单结构兼容。
6. 所有公开接口必须包含 Swagger 注释。

### 本阶段只保留 Health

- `application`
- `im`
- `app`

这些模块不得预建未经确认的业务实体。

## 验收标准

1. `frontend/` 与 `backend/` 分层清楚，后端按模块组织。
2. MySQL 8.4 空库可以自动完成初始化。
3. `go test ./...`、Swagger 生成、Go 构建和前端生产构建通过。
4. 后端与前端正常启动。
5. `admin / admin123` 可在默认租户登录。
6. 首页无 404 或服务器错误提示。
7. 已显示的菜单均有真实可用的后端接口；未迁移完成的菜单不得提前暴露。
8. Java 与 Go 的接口差异通过清单持续收敛，不以空壳接口冒充迁移完成。

## 实施顺序

1. 调整工程目录并建立平台层、模块层边界。
2. 迁移 System 最小闭环：认证、权限、菜单、字典、用户、角色、部门与租户。
3. 迁移 Infra、Member、Pay 的管理端闭环。
4. 补齐审计、任务、文件与外部支付通道的可选适配。
5. 将同一模块实现复用于 Go Cloud，并按服务进程拆分。

## 当前已验证闭环

- System：租户识别、登录/JWT、权限菜单、字典、通知入口、运营用户 CRUD、部门/岗位简表。
- Infra：参数配置 CRUD、文件配置 CRUD/主配置切换、API 访问日志分页。
- Member：会员 CRUD/状态/等级/积分调整，等级、分组、标签 CRUD，积分流水事务。
- Pay：支付应用与渠道 CRUD，订单创建/查询，退款创建/查询/撤销及金额事务约束。
- Application、IM、App：只提供 Health，不建立业务表。
- Go 测试、Swagger、后端构建、前端类型检查与生产构建，以及启动、初始化、登录和已展示菜单接口均已通过。

未在菜单中开放的 Java 扩展能力（例如定时任务、对象存储真实上传、第三方支付适配、钱包/转账）仍作为后续按需迁移项，当前不以空接口冒充完成。
